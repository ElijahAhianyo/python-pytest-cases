104@function_decorator 

105def cases_generator(names=None, # type: Union[str, Callable[[Any], str], Iterable[str]] 

106 # target=None, # type: Any 

107 # tags=None, # type: Iterable[Any] 

108 lru_cache=False, # type: bool, 

109 case_func=DECORATED, # noqa 

110 **param_ranges # type: Iterable[Any] 

111 ): 

112 """ DEPRECATED - use `@parametrize(**<param_ranges>, idgen=<names>)` instead 

113 

114 Note that `@cases_generator` (legacy api) does not work correctly with `@parametrize_with_cases` (new api). Only 

115 `@cases_data` (legacy api) is compliant. See https://github.com/smarie/python-pytest-cases/issues/124 

116 

117 Decorator to declare a case function as being a cases generator. `param_ranges` should be a named list of parameter 

118 ranges to explore to generate the cases. 

119 

120 The decorator will use `itertools.product` to create a cartesian product of the named parameter ranges, and create 

121 a case for each combination. When the case function will be called for a given combination, the corresponding 

122 parameters will be passed to the decorated function. 

123 

124 >>> @cases_generator("test with i={i}", i=range(10)) 

125 ... def case_10_times(i): 

126 ... ''' Generates 10 cases ''' 

127 ... ins = dict(a=i, b=i+1) 

128 ... outs = i+1, i+2 

129 ... return ins, outs, None 

130 

131 :param names: a name template, that will be transformed into the case name using 

132 `names.format(**params)` for each case, where `params` is the dictionary of parameter values for this 

133 generated case. Alternately a callable returning a string can be provided, in which case 

134 `names(**params)` will be used. Finally an explicit list of names can be provided, in which case it should have 

135 the correct length (an error will be raised otherwise). 

136 :param lru_cache: a boolean (default False) indicating if the generated cases should be cached. This is identical 

137 to decorating the function with an additional `@lru_cache(maxsize=n)` where n is the total number of generated 

138 cases. 

139 :param param_ranges: named parameters and for each of them the list of values to be used to generate cases. For 

140 each combination of values (a cartesian product is made) the parameters will be passed to the underlying 

141 function so they should have names the underlying function can handle. 

142 :return: 

143 """ 

144 warn("`@case_generator` is deprecated. Please use `@parametrize(**<param_ranges>, idgen=<names>)` instead.", 

145 category=DeprecationWarning, stacklevel=2) 

146 

147 kwarg_values = list(product(*param_ranges.values())) 

148 setattr(case_func, _GENERATOR_FIELD, (names, param_ranges.keys(), kwarg_values)) 

149 

150 if lru_cache: 

151 nb_cases = len(kwarg_values) 

152 # decorate the function with the appropriate lru cache size 

153 case_func = lru(maxsize=nb_cases)(case_func) 

154 

155 return case_func
